Zagadnienie mostw krlewieckich

  • zagadnienie, nad ktrym rzekomo gowili si mieszkacy Krlewca
  • rozwizane przez Leonharda Eulera (w Solutio problematis ad geometriam situs pertinentis w Commentarii academiae scientiarum Petropolitanae, 1741)
  • pierwsza praca z teorii grafw

  • czy mona przej kolejno przez wszystkie mosty tak, eby kady przekroczy tylko raz?

  • odpowied brzmi: nie
    • decyduje o tym nieparzysta liczba wylotw mostw zarwno na kad z wysp, jak i na oba brzegi rzeki
  • graf eulerowski - graf spjny, ktry mona opisa lini cig w taki sposb, aby kada krawd tego grafu bya obwiedziona tylko raz (tzw. cykl Eulera)
    • moliwe tylko wtedy, gdy liczba wierzchokw tego grafu, w ktrych spotyka si nieparzysta liczba krawdzi, wynosi 0 lub 2
    • aby przej wszystkie krawdzie grafu i wrci do punktu wyjcia, nie moe on zawiera adnych wzw, w ktrych spotyka si nieparzysta liczba krawdzi

Pojcia podstawowe

Wze (wierzchoek)

  • podstawowy element skadowy grafu
  • w teorii grafw - punkt pewnej przestrzeni (zbioru) VV, nad ktr zbudowany jest graf
  • reprezentuje pewien obiekt
  • moe posiada nazw (klucz) oraz dodatkowe dane

Krawd

  • podstawowy element skadowy grafu
  • w teorii grafw - para wyrnionych wierzchokw grafu, czyli takich, ktre s ze sob poczone (ssiednie)
  • reprezentuje relacje midzy obiektami
  • moe by jedno- lub dwukierunkowa

Graf (graf prosty lub nieskierowany)

Graf nieskierowany GG skada si z dwch zbiorw VV EE, przy czym VV jest niepustym zbiorem, ktrego elementy nazywane s wierzchokami, a EE jest rodzin dwuelementowych podzbiorw zbioru wierzchokw VV, zwanych krawdziami,

E{{u,v}:u,vV,uv}E{{u,v}:u,vV,uv}

Graf skierowany

Graf skierowany (digraf) skada si z dwch zbiorw niepustego zbioru wierzchokw VV oraz rodziny AA par uporzdkowanych elementw zbioru VV, zwanych krawdziami lub ukami grafu skierowanego. Kolejno wierzchokw w parze wyznacza kierunek krawdzi w przypadku pary v,uv,u uk biegnie z wierzchoka vv do wierzchoka uu.

Waga

  • wartoci przypisane krawdziom
  • mog okrela np.:
    • koszt przejcia midzy wierzchokami
    • przepustowo poczenia
    • czstotliwo kontaktw

cieka

  • sekwencja wierzchokw poczonych krawdziami
w1,w2,...,wn,  (wi,wi+1)E,  1in1w1,w2,...,wn,  (wi,wi+1)E,  1in1

  • waona dugo cieki to suma wag wszystkich krawdzi skadajcych si na ciek

Droga

  • cieka, w ktrej wierzchoki s rne (z wyjtkiem ewentualnej rwnoci wierzchokw pierwszego i ostatniego mamy wtedy do czynienia z tzw. cyklem)

  • droga zamknita, czyli taka, ktrej koniec (ostatni wierzchoek) jest identyczny z pocztkiem (pierwszym wierzchokiem)

  • w naturalny sposb uwzgldnia wagi krawdzi
  • dla maych grafw dua przejrzysto - od razu wida, ktre wierzchoki s ze sob poczone
  • w powyszym przykadzie wikszo komrek jest pusta:
    • niezbyt wydajne zarzdzanie pamici w przypadku grafw "rzadkich"

  • zoono typowych operacji:
    • wstawianie krawdzi: O(1)O(1)

class Vertex:
    def __init__(self,key):
        self.id = key
        self.connectedTo = {}   #lista ssiedztwa z wagami

    def addNeighbor(self,nbr,weight=0):
        self.connectedTo[nbr] = weight

    def __str__(self):
        return str(self.id) + ' connectedTo: ' + str([x.id for x in self.connectedTo])

    def getConnections(self):
        return self.connectedTo.keys()

    def getId(self):
        return self.id

    def getWeight(self,nbr):
        return self.connectedTo[nbr]

class Graph:
    def __init__(self):
        self.vertList = {}
        self.numVertices = 0

    def addVertex(self,key):
        self.numVertices = self.numVertices + 1
        newVertex = Vertex(key)
        self.vertList[key] = newVertex
        return newVertex

    def getVertex(self,n):
        if n in self.vertList:
            return self.vertList[n]
        else:
            return None

    def __contains__(self,n):
        return n in self.vertList

    def addEdge(self,f,t,cost=0):
        if f not in self.vertList:
            nv = self.addVertex(f)
        if t not in self.vertList:
            nv = self.addVertex(t)
        self.vertList[f].addNeighbor(self.vertList[t], cost)

    def getVertices(self):
        return self.vertList.keys()

    def __iter__(self):
        return iter(self.vertList.values())

Wyszukiwanie wszerz

Studium przypadku - drabina sowna (ang. word ladder problem)

  • zabawa sowna, wymylona w 1878 roku przez Lewisa Carolla (autora "Alicji w krainie czarw")
  • trzeba przeksztaci podane sowo, np. FOOL w inne sowo, np. SAGE
  • przeksztace dokonuje si krok po kroku
  • w kadym kroku mona zmieni tylko jedn liter
  • dopuszczalne s jedynie zmiany w istniejce sowa, np. 
    FOOL
POOL
POLL
POLE
PALE
SALE
SAGE
  • istnieje wiele wariantw, np. transformacja:
    • w okrelonej liczbie krokw
    • w najmniejszej liczbie krokw
    • przy uyciu okrelonych sw
  • nasz cel - transformacja w najmniejszej liczbie krokw
  • sposb postpowania:
    • reprezentacja powiza midzy sowami w formie grafu
    • skorzystanie z algorytmu wyszukiwania wszerz (ang. breadth first search, BFS) do znalezienia najkrtszej cieki w grafie

Budowanie grafu

  • krawd midzy kad par sw, ktre rni si tylko jedn liter

  • dysponujc takim grafem, kada cieka midzy FOOL a SAGE jest rozwizaniem zagadki
  • najkrtsza z nich rozwizuje dodatkowo nasze zadanie
  • zamy, e wszystkie sowa na licie s tej samej dugoci
  • podejcie pierwsze:
    • moemy porwna kade sowo na licie ze wszystkimi pozostaymi i znale w ten sposb te rnice si tylko jedn liter
    • operacja klasy O(n2)O(n2)
    • w porzdku dla niewielkich zbiorw sw
    • jeli sw mamy np. 3130, trzeba wykona prawie 10 milionw porwna
    • mao wydajne
  • podejcie drugie
    • zamy, e mamy wiele kubekw, ktrych nazwy powstay ze sw, w ktrych jedna litera zostaa zastpiona znakiem podkrelenia
    • traktujc ten znak jako dzik kart, moemy sprawdza, ktre ze sw pasuje do odpowiedniego kubeka
    • wszystkie sowa w jednym kubeku s ze sob poczone
    • operacja klasy O(n)O(n)

def buildGraph(wordFile):
    d = {}
    g = Graph()
    wfile = open(wordFile,'r')
    # create buckets of words that differ by one letter
    for line in wfile:
        word = line[:-1]
        for i in range(len(word)):
            bucket = word[:i] + '_' + word[i+1:]
            if bucket in d:
                d[bucket].append(word)
            else:
                d[bucket] = [word]
    # add vertices and edges for words in the same bucket
    for bucket in d.keys():
        for word1 in d[bucket]:
            for word2 in d[bucket]:
                if word1 != word2:
                    g.addEdge(word1,word2)
    return g

def bfs(g,start):
    start.setDistance(0)                            #distance 0 indicates it is a start node
    start.setPred(None)                             #no predecessor at start
    vertQueue = Queue()
    vertQueue.enqueue(start)                        #add start to processing queue
    while (vertQueue.size() > 0):
        currentVert = vertQueue.dequeue()           #pop next node to process -> current node
        for nbr in currentVert.getConnections():    #check all neighbors of the current node
            if (nbr.getColor() == 'white'):         #if the neighbor is white
                nbr.setColor('gray')                             #change its color to grey
                nbr.setDistance(currentVert.getDistance() + 1)   #set its distance
                nbr.setPred(currentVert)                         #current node is its predecessor
                vertQueue.enqueue(nbr)                           #add it to the queue
        currentVert.setColor('black')               #change current node to black after visiting all of its neigh.
  • przeledmy dziaanie algorytmu krok po kroku

    print(x.getId())
